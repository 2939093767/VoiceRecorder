# VoiceRecorder 语音记录应用

## 1 项目概述

### 1.1 产品定位

VoiceRecorder 是一款基于 Qt 6.5.3 + C++17 构建的桌面端语音记录应用，采用分层架构设计，遵循 openqt 框架准则。

### 1.2 核心功能

| 功能模块 | 说明 |
|----------|------|
| 语音录制 | 本地麦克风音频实时采集 |
| 语音识别 | Windows SAPI 引擎，实时转文字 |
| 会议总结 | 云端大模型 AI 智能总结 |
| 记录管理 | JSON 本地持久化，搜索过滤 |
| 抽屉式导航 | 页面切换管理 |

## 2 技术架构

### 2.1 分层架构图

```
┌─────────────────────────────────────────────────────┐
│                   顶层 UI 层                          │
│                  CentralWindow                       │
│            主窗口布局 │ 页面切换 │ 样式加载          │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│              SignalPage 子页面管理层                   │
│    PageBase │ VoiceRecorderPage │ MeetingSummary   │
│          承接参数 │ 转发调用 │ 回显结果              │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│                Ability 业务能力层                     │
│    RecordManager │ AudioRecorder │ SpeechRecognizer │
│           CloudLLM  (skill_Ability 命名空间)         │
│            纯后台逻辑 │ 无 UI 依赖 │ 业务封装        │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│                  models 数据模型层                    │
│                   RecordItem                         │
└─────────────────────────────────────────────────────┘
```

### 2.2 分层职责

| 层级 | 目录 | 职责范围 | 约束 |
|------|------|----------|------|
| 顶层 UI | `centralwindow.cpp` | 窗口布局、页面切换、样式加载 | 仅调用 SignalPage 接口 |
| SignalPage | `SignalPage/` | 参数转发、UI 回显、信号槽连接 | 不编写业务逻辑 |
| Ability | `Ability/` | 纯业务逻辑、线程调度、数据处理 | 零 UI 依赖 |
| models | `models/` | 数据结构定义 | 仅数据结构 |
| CustomControl | `CustomControl/` | 通用 UI 组件封装 | 不含业务逻辑 |

### 2.3 调用链路

```
用户交互
    ↓
顶层 UI (CentralWindow)
    ↓
SignalPage (页面层) ──转发──▶ Ability (业务层)
    ↑                              │
    │                              ▼
    └─────── 返回结果 ────────── 业务执行
```

**分层禁止事项：**
- 禁止跨层逆向调用
- 禁止在 UI 层、SignalPage 层直接访问 Ability 实现类内部方法
- 禁止在 Ability 层使用任何 Qt UI 控件

## 4 Ability 业务能力层

### 4.1 设计原则

- 纯后台逻辑，无任何 Qt UI 控件引用
- 统一纳入 `skill_Ability` 命名空间
- 仅对外暴露公开接口，禁止暴露内部实现细节
- 线程创建、调度统一在 Ability 层完成

### 4.2 模块清单

| 模块 | 文件 | 核心职责 |
|------|------|----------|
| RecordManager | recordmanager.h/cpp | 记录增删改查、JSON 持久化、搜索 |
| AudioRecorder | audiorecorder.h/cpp | 麦克风采集、音量计算、时长统计 |
| SpeechRecognizer | speechrecognizer.h/cpp | Windows SAPI 语音识别 |
| CloudLLM | cloudllm.h/cpp | HTTP 调用大模型 API、SSE 流式输出 |

### 4.3 RecordManager 接口

```cpp
namespace skill_Ability {

class RecordManager : public QObject {
    Q_OBJECT
public:
    QList<RecordItem> allRecords() const;           // 获取所有记录
    RecordItem recordById(const QString &id) const;  // 根据 ID 获取
    QString addRecord(const RecordItem &record);     // 添加记录
    bool updateRecord(const QString &id, const RecordItem &record); // 更新
    bool deleteRecord(const QString &id);            // 删除记录
    QList<RecordItem> searchRecords(const QString &keyword) const; // 搜索
    bool loadFromDisk();                             // 加载数据
    bool saveToDisk() const;                         // 保存数据

signals:
    void recordsChanged();                           // 记录变化信号
};

}
```

## 5 SignalPage 子页面管理层

### 5.1 设计原则

- 承接 UI 参数、转发业务调用、接收返回结果
- 不编写任何业务逻辑
- 不直接修改控件属性（通过 UI 文件配置）
- 不绑定、更改控件信号槽连接

### 5.2 PageBase 页面基类

所有子页面必须继承此类：

```cpp
class PageBase : public QWidget {
    Q_OBJECT
public:
    virtual QString pageName() const = 0;       // 页面名称
    virtual void onPageActivated();             // 激活时调用
    virtual void onPageDeactivated();           // 停用时调用
    virtual void setStatusMessage(const QString &);

signals:
    void requestSwitchPage(const QString &pageName);  // 请求切换页面
    void statusMessageChanged(const QString &);      // 状态消息变化
};
```

### 5.3 VoiceRecorderPage 接口

```cpp
class VoiceRecorderPage : public PageBase {

public:
    RecordItem currentRecord() const;           // 当前编辑记录
```

## 7 UI 设计规范

### 7.1 样式管理

```
styles/
└── main.qss   # 全局样式表（统一加载）
```

**样式约束：**
- 全局样式通过 main.qss 统一加载
- 局部微调允许使用 setStyleSheet
- h/cpp 代码禁止硬编码样式属性

### 7.2 配色规范（暗色主题）

| 类型 | 色值 | 用途 |
|------|------|------|
| 背景基色 | #1e1e1e | 主窗口背景 |
| 卡片色 | #2d2d2d | 子页面/卡片 |
| 主文本 | #e0e0e0 | 主要文字 |
| 次文本 | #a0a0a0 | 次要文字 |

### 7.3 UI 开发红线

- 禁止通过 ui 文件手动设置控件样式
- 禁止在代码中硬编码颜色、字体、尺寸
- 禁止随意修改暗色主题基调

## 8 线程设计规范

### 8.1 线程层级

| 层级 | 线程权责 |
|------|----------|
| Ability | 唯一线程管理层，创建、启动、终止线程 |
| SignalPage | 不创建线程，仅接收结果 |
| 顶层 UI | 不参与线程调度 |

### 8.2 线程禁止事项

- 禁止在 UI 层、SignalPage 层创建线程
- 禁止子线程操作 Qt UI 控件
- 禁止使用 std::thread 等非 Qt 原生线程库

## 11 编译指南

### Qt Creator
1. 打开 `VoiceRecorder.pro`
2. 选择「Desktop Qt 6.5.3 MSVC2019 64bit」套件
3. 执行「重新运行 qmake」
4. 执行「构建」

### 命令行
```bash
mkdir build && cd build
qmake ..\VoiceRecorder.pro -spec win32-msvc CONFIG+=release
nmake
```
