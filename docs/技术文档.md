# VoiceRecorder 语音记录应用 - 技术文档

## 一、产品简介

VoiceRecorder 是一款基于 Qt 6.5.3 开发的桌面端语音记录应用，旨在为用户提供便捷的语音转文字记录体验。应用通过接入本地麦克风进行实时语音采集，并利用系统内置语音识别引擎将语音内容实时转换为文字，同时支持自由文本备注编辑和历史记录管理。

应用采用现代化扁平化设计风格，界面简洁直观，左侧为历史记录列表，右侧为主要编辑区域，用户可以快速创建、查看和管理语音记录。

---

## 二、核心功能概括

### 2.1 语音录制与识别
- 本地麦克风音频实时采集
- 系统级语音识别引擎集成
- 实时音量指示器和录音时长显示
- 语音识别结果实时追加到文本编辑器

### 2.2 文字记录与编辑
- 标题编辑（占位符显示"新建会话"）
- 语音识别内容编辑区
- 自由备注编辑区
- 未保存更改提示机制

### 2.3 记录管理
- 记录自动保存到本地
- 历史记录列表展示
- 新建、删除、切换记录
- 记录创建时间和时长追踪

### 2.4 界面设计
- 现代化扁平化 UI 设计
- 外部 QSS 样式表管理
- 响应式布局
- 沉浸式编辑体验

### 3.2 架构说明

应用采用经典的 Qt Widgets 单窗口架构，以主窗口为核心，内部划分为三大功能模块：

- **界面层**：主窗口及其内部控件，负责用户交互和数据展示
- **功能模块层**：音频录制、语音识别、数据存储三个独立模块
- **数据层**：本地 JSON 文件持久化存储

模块之间通过信号与槽机制进行通信，实现低耦合高内聚的设计目标。

## 五、模块功能说明

### 5.1 主窗口模块

主窗口模块是应用的核心，负责整体 UI 布局、用户交互处理、状态管理和模块协调。

**主要功能：**
- 左右分栏式布局管理
- 记录状态追踪（已保存/未保存）
- 页面切换逻辑控制
- 全局信号调度

**技术亮点：**
- 使用 QSS 外部样式表实现主题化管理，全局样式与代码逻辑完全分离
- 采用信号与槽机制实现模块间松耦合通信
- 未保存更改检测与提示机制，防止用户数据丢失

### 5.2 音频录制模块

音频录制模块负责麦克风音频采集、音量检测和录音时长统计。

**主要功能：**
- 麦克风设备初始化与管理
- 实时音频数据采集
- 音量级别计算
- 录音时长统计

**技术亮点：**
- 基于 Qt Multimedia 的跨平台音频采集方案
- 16kHz 单声道 16位 PCM 格式，专为语音识别优化
- 实时音量反馈，提供直观的录音状态指示

### 5.3 语音识别模块

语音识别模块负责将音频流实时转换为文字内容。

**主要功能：**
- 语音识别引擎初始化
- 实时听写模式
- 识别结果回调处理
- 错误状态处理

**技术亮点：**
- 集成 Windows 系统级 SAPI 语音识别引擎，无需云端服务
- 支持中间结果（Hypothesis）实时反馈
- 最终结果精确追加，保证文本准确性

### 5.4 数据持久化模块

数据持久化模块负责记录数据的本地存储与读取。

**主要功能：**
- 记录数据序列化/反序列化
- JSON 文件读写
- 历史记录列表维护
- 自动保存机制

**技术亮点：**
- 采用 JSON 格式存储，便于数据迁移和解析
- 用户目录下独立存储路径，数据安全隔离
- 应用退出时自动保存，防止数据丢失

## 九、数据存储路径

应用数据存储在用户主目录下：

```
%USERPROFILE%/VoiceRecorder/
└── records.json    # 记录数据文件
```

所有语音记录以 JSON 格式存储，包含标题、内容、备注、创建时间和时长等信息。
